Guía de Despliegue: mi-nuevo-proyecto

Sigue estos pasos para conectar este nuevo proyecto al flujo automatizado y exponerlo en internet.

1 Preparar el Repositorio (Código Fuente)

Asegúrate de incluir estos dos archivos clave en la raíz de tu proyecto en GitHub:

A. El archivo docker-compose.yml

Presta atención al container_name y al puerto interno (los usaremos en NPM). Para no exponer la app directamente al host, la conectaremos a la red externa red_app.
Nota: Usualmente tu aplicación es quien expone el puerto internamente, no hace falta declararlo explícitamente con "expose" o "ports" en este archivo a menos que sea necesario.

B. El archivo Jenkinsfile

Copia y pega este código en un archivo llamado Jenkinsfile. Contiene la ruta exacta a donde Jenkins debe ir en tu servidor.

2 Configurar GitHub Webhook

Abrir GitHub

Dentro del proyecto de GitHub, ve a Settings > Webhooks > Add webhook en tu repositorio mi-nuevo-proyecto.

  • Activa Just the push event
  • Content type: application/json
  • Payload URL: Copia la siguiente ruta (¡Incluye la barra al final!):

3 Preparar el Servidor Ubuntu (Solo primera vez)

Vamos a conectarnos al servidor y a clonar el proyecto usando el alias para finalmente crear el archivo .env de forma segura.

1. Conéctate al servidor por SSH

2. Entra a la ruta base

3. Clona el proyecto

4. Entra a la carpeta del proyecto

5. Crea el archivo de variables seguras

nano .env

📝 Instrucciones para el editor Nano:

Una vez que se abra el editor, pega todas tus credenciales (las mismas que usa tu app). Por ejemplo:

PORT=3000
DB_HOST=localhost
API_KEY=tu_clave_super_secreta

Para salir y guardar presiona: Ctrl + O, luego Enter, y finalmente Ctrl + X.
¡Eso es todo! No ejecutes docker compose up, Jenkins lo hará por ti.

4 Crear la Tarea en Jenkins

Abrir Jenkins

1. Entra a Jenkins > Nueva Tarea > Nómbrala > Selecciona Pipeline.

Pestaña General:
Marca "GitHub project" y pega la URL web:

Marca la option GitHub hook trigger for GITScm polling.
Sección Pipeline:
• Definition: Pipeline script from SCM
• SCM: Git
• Repository URL:
• Credentials: Selecciona el Token de
• Branches to build: */main

5 Arranque Manual (Anclaje)

Para que Jenkins vincule el Webhook recién creado, debes iniciar la tarea manualmente una sola vez:

  1. Haz clic en "Construir ahora" (Build Now) en Jenkins.
  2. Verifica que termine con estado SUCCESS.
  3. A partir de ahora, cualquier git push desplegará tu app automáticamente. 🚀
Paso Final

6 Exponer con Nginx Proxy Manager

Configura el proxy inverso para asociar tu dominio a la aplicación, habilitar HTTPS y publicarla al mundo. (el link al administrador de NPM solo funciona desde la red local)

Abrir Nginx Proxy Manager

Pestaña "Details"

Añade un New Proxy Host y llena los datos exactos:

  • Domain Names:
  • Scheme: http
  • Forward Hostname / IP: (¡Importante! Es el nombre exacto del contenedor)
  • Forward Port: (Puerto interno de la app, NO el mapeado en host)
  • Activar opción: Block Common Exploits

Pestaña "SSL"

Genera el certificado automático (Let's Encrypt ya está preconfigurado):

  • SSL Certificate:
    Selecciona el desplegable y elige:
    "Request a new SSL Certificate"
  • Activar opción: Force SSL
  • Activar opción: HTTP/2 Support
  • Nota: Tu correo ya está en el sistema, solo asegúrate de marcar "I Agree to the Let's Encrypt Terms of Service" y presiona Save.

7 ¡Prueba tu Aplicación! 🎉

Si todos los pasos anteriores fueron exitosos y Jenkins terminó la tarea, tu aplicación ya debería estar en línea y protegida con SSL.
